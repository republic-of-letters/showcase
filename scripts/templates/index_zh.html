你是哪一种人？

「我有数据」

数据从不移动。代码上门，你来运行，只有聚合结果回去。

  1. 读一读「数据方 × 想法方」模式
  2. 在布告栏登一张「提供」——只登剪影，不登 schema
  3. 有人敲门时，以执行方身份开项目

「我有想法，没数据」

这里可能正有一份数据，在等你这个问题。

  1. 扫一遍布告栏和项目
  2. 应答一张布告，或申请加入
  3. 提出一个 round——代码走过去，你不需拿到数据

「我是学生」

在护栏内做真实研究——每份贡献都永久署你名下。

  1. 挑一个开放申请的项目
  2. 带代理入职，通过演练（约 30 分钟）
  3. 提出 round；导师共签每次合并

「我们已有团队」

跳过相识，直接拿走整套机制。

  1. gh repo create … --template republic-of-letters/protocol --private
  2. 十分钟走完 SETUP.md
  3. 开第一个课题；每位成员演练一次

为什么这样做研究

跨机构研究卡在四件事：数据传不出、代码不敢跑、过程没记录、署名谈不拢。 这个空间从结构上把四件都解决。

数据

数据从不离开你的机器

合作者寄来代码；你审查后自己运行；回来的只有图、表、文字结论。CI 物理拦截 任何进入仓库的原始数据。

代理

为智能体时代而设计

每位成员都带 AI 代理工作。协议就是写给代理执行的——代理跑腿，人保留每个 要紧的决定。

记录

全程留痕，包括失败

每个问题、每次运行、每条死路都是带版本的文件。新合作者——或新开的代理会话—— 都能重建完整历史。

信任

信任来自结构，不靠人品

代码触碰数据前，必过自动扫描加人工阅读；结果进档案前，须双方点头。每个数字 都带来源等级。

署名

署名先小人后君子

作者排序在课题 GO 之前白纸黑字写下，数据与算力都计入。拖着不谈是违反协议。

成本

零基础设施，零锁定

就是普通 GitHub：免费、熟悉、随时带走。一条命令开项目；流程里没有任何私有 依赖。

支持哪些模式

一套协议，多种形态——同一套机制，为不同的合作方式排布。

合著论文

一个课题 = 一篇候选论文。试探 → GO → 分析 round → 合并档案，就是这篇论文 完整的审计线索。

适合：2–5 位有共同问题的研究者。

数据方 × 想法方

一方有数据和算力，另一方有假设和代码。受限授权数据留在原地，只在衍生表层面 连接。

适合：数据持有者 + 方法研究者。

课题 / 研究计划

一个立项课题或长期议程 = 一个仓库、多个 topic。决策日志是它的记忆，里程碑是 可核查的 round。

适合：基金课题、实验室、多篇论文合作。

学位论文与培养

学生提出 round，导师共签每次合并。护栏内做真实研究，留下最干净的成长记录。

适合：导师与研究生、本科生。

复现与稳健性

把已有分析独立重跑。成功复现的结果获得 replicated 来源等级—— 最理想的第一份贡献。

适合：新成员；投稿前焊牢结果的团队。

运作方式

一 · 课题

一篇候选论文，一个课题

假设、证伪条件、署名、决策日志都在一个文件里。杀掉课题是正式产出，附带 尸检报告。

二 · ROUND

一个问题，一个 Pull Request

提议方写下问题和可运行的代码；数据方运行后，把聚合结果寄回同一线程。

三 · 人闸

三扇只有人能开的门

课题 GO/终止、触碰数据的代码、合并。代理起草，人做决定。任何人都能用一个 标签叫停一个 round。

项目

研究在私有仓库里进行；卡片只展示各项目批准公开的部分。 仅展示＝只看不进；开放申请＝有一扇门，去敲。

{{CARDS}}

布告栏

合作始于撮合：有人握着数据，有人握着这份数据配得上的问题。 提供（我有数据、授权、算力、方法）与征求（我需要它们）。

布告只登剪影。表单粗粒度是刻意的——这上面没有能被 抄走的东西，只有能被认出的轮廓。细节沿用数据本身的边界；每张布告都有有效期， 到期自动下架。

{{NOTICES}}

登一张：成员对 notices.yaml 提 PR；其他人用布告表单。 应答：直接联系留的 handle，或开 issue 写明布告编号——匿名发布者 在暗处看着，会主动找你。

如何参与

  1. 先逛逛

    看看项目和布告栏，然后读一封信——五分钟就知道在这里工作 是什么感觉。

  2. 选一扇门

    申请加入项目， 或提议一个：一个问题、一个证伪条件、一份数据计划、至少一位确定的合作者。

  3. 入职——你和你的代理

    指南把你从没有账号带到可以工作，写给你的 AI 代理执行。结尾是一次演练： 完整走一遍练习 round。

  4. 以 round 工作

    提出代码；结果寄回同一线程。迭代、争论、收敛——一切落在文字上。

  5. 在闸口决策

    人判 GO、批准运行、决定合并。通过全部三道闸的成果进入档案——你愿意的话， 也成为这里的一张卡片。

可以相信什么

这里的信任是流水线的属性，不是口头承诺。每个公开产物都带来源等级：

等级含义

常见问题

我需要会 git 或 GitHub 吗？

基本不用。入职指南写给你的代理执行；你只需注册账号、点几下。会收发邮件就能 在这里工作。

我的数据安全吗？

数据从不离开你的机器。每次分析都由你亲自运行，运行前先扫描加人工读码——沙箱、 只读、无网络。仓库只承载聚合结果。

想法被人拿走怎么办？

项目是私有信任圈——发表前圈外什么都看不见。圈内每个假设、每个 round 都带时间戳， 优先权可证明。布告栏只有剪影，上面没有值得偷的东西。

中途要退出怎么办？

退出是被记录的决定。已合并的贡献永久署你名下，GO 时的署名规则继续有效，你还能 带走一份完整档案副本。

需要什么 AI 代理？

任何有编码能力的代理——Claude Code、Codex 等。协议与具体代理无关。没有代理也能 参与，只是跑腿自己干。

署名归谁？

归人。作者排序在课题 GO 之前书面约定，数据与算力都计入。代理是笔，不是作者。

谁能看到我的项目？

你自己选：仅展示、开放申请、或完全开放。研究仓库始终 只对受邀成员可见，除非你主动公开。

要花钱吗？

不用。普通 GitHub——私有研究仓库免费——协议本身开源（MIT）。
